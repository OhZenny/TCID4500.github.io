    • Pull Requests and code reviews within VSCode

    Pull requests are way of asking permission to add changes to the main code. Others can review and discuss the changes before actually adding them to the main branch. It's a way to make sure everything works well together and doesn't break anything.

    Install the extension GitHub Pull Requests open_in_new to be able to make pull requests seamlessly in VSCode.

    Review and discuss pull requests

    1. In VSCode, make changes to a document that is connected to a repo. on VSCode.

    2. Commit those changes.

    3. Navigate to the GitHub Pull Request on the Primary Side Bar. >"

    4. Choose which base you would like to merge your working branch with.

    5. Choose a relevant title for this request.

    6. Write a description about why and what changes you made.

    7. Click "Create". A page will be created that describes the pull request.

    Create pull requests using GitHub and VS Code

    1. Reviewers can "Checkout" to view a side-by-side comparison with the original and changes.

    2. The pull request and the discussions inside of it will be seen on the GitHub repository site under "Pull Requests".

    3. Reviews can approve the changes and merge the branches together.
