    • Branching with VSCode

    Visual Studio Code (VSCode) makes it easy to create and switch between these branches. Think of branches like a guide that shows you the different roads you can take for your work. This way, you can work on your part of the project without interfering with what others are doing.

    Branching is a good way for developers to work on their tasks independently and then bring everything together when they're ready.

    Use VS Code to create and switch between branches.

    Here's how you can create and switch branches:

    Creating

    1. In the status bar at the bottom right, click the source control icon with the name of the branch besides it. It should say "main" or "master" or whatever you named your origin branch.

    2. Select "Create a new branch".

    3. Choose what you'd like to name this new branch. You can read more about naming branches on Tilburg Science Hub's open_in_new article on the topic.

    Switching

    Once you've created a new branch, you will automatically start working in it. You can do the following to switch to different branches:

    1. In the status bar at the bottom right, click the source control icon with the name of the branch besides it. It should say "main" or "master" or whatever you named your origin branch.

    2. Select the desired branch.

    Understanding the concept of branching for feature development or bug fixes.

    When working on a big project, its a best practice to create branches with a specific focus. Typically, this will be a feature or a bugfix.

    Why is this imporant? It keepsthe project organized and creates an easier to understand roadmap. Without branches, making changes directly to the main branch could lead to chaos, confusion, and errors.

    Commiting to particular branches makes reconciling and merging code more managable.
