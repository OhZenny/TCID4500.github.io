    • Setting Up a Git and Visual Studio Code Environment

    First things first, install the most recent versions of Git and VS Code. This guide was written with Git version 2.43.0, so things might look different from this guide, but the steps should be similar.

    Installing Git

    Visit Git’s website and head to the downloads page or click the link here open_in_new .

    Download the correct setup for your operating system by following the instructions on their page.

    1.Keep the default checks on the first page

    2.Change the default editor to “Use Visual Studio Code as Git’s default editor”.

    3.Choose "Override the default branch name for new repositories". Rename it to "main".

    4.Choose "Git from the command line and also from 3rd-party software".

    5.Choose "Use bundled openSSH".

    6.Choose "Use the OpenSSL library".

    7.Choose "Checkout Windows-style, commit Unix-style line endings".

    8.Choose "MinTTY".

    9.Choose "Fast-fordward or merge”.

    10.Choose "Git Credential Manager”.

    11.Check "Enable file system caching" and leave "Enable symbolic links" unchecked.

    12.Leave both boxes unchecked. Click "Install".

    Installing Visual Studio Code

    Visit Visual Studio Code's (VSCode) website and head to the downloads page or click the link here open_in_new . Make sure to select the download that corresponds to your operating system.

    1.Read through and accept the license agreement.

    2.Customize further installation settings, or leave default.

    3.Click "install".

    Configuring Git with VSCode

    Visual Studio Code (VSCode) allows developers to utilize a Git repository. In order to get this started inside VScode, we need to have Git already installed. Once you've done that, restart VSCode.

    1.With VSCode open, go to the top app ribbon. Locate "Terminal".

    2.Under "Terminal", click "New Terminal". The terminal will open at the bottom of VSCode as a window.

    3.In the terminal, type the following line (replace placeholders with your own): 

                    git config --global user.email youremail.@google.com
            
                    git config --global user.name yourusername

    4.Check to see if your email and username applied with the following code. The output should be the username and email that you just set. 

                git config --global user.email
        
                git config --global user.name

    Visit Visual Studio Code's (VSCode) documents for more indepth information. here open_in_new .
